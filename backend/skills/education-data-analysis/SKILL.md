# 教育学情数据分析 Skill

本技能专门针对教育学情数据进行深度分析，包括学生成长环境、学生发展指标、群体差异识别、学校层面定位、四象限分层干预等核心分析模块。

## 适用场景

- 学生指标得分数据分析
- 教育质量评估报告
- 学情画像构建
- 学生分层干预方案制定
- 学校教育质量对比分析

## 数据要求

### 数据格式
- Excel文件 (.xlsx, .xls)
- CSV文件 (.csv)
- JSON文件 (.json)

### 数据结构要求
数据应包含以下维度（列名可灵活配置）：
- **基本信息**：学校代码、年级、班级、考号、姓名、性别
- **办学属性**：办学规模、办学性质
- **成长环境指标**：亲子关系、师生关系、同伴关系、校园安全等
- **学生发展指标**：身心健康、情绪状态、运动健康、学习创新机会、学习习惯、学业达标等

### 示例数据结构
```
办学规模 | 办学性质 | 学校代码 | 年级 | 班级 | 考号 | 姓名 | 性别 | 
成长环境_亲子关系 | 成长环境_师生关系 | 成长环境_同伴关系 | 成长环境_校园安全 |
学生发展_身心健康 | 学生发展_情绪状态 | 学生发展_运动健康 | 
学生发展_学习创新机会 | 学生发展_学习习惯 | 学生发展_学业达标
```

## 分析流程

### 1. 数据概况与分析口径
- 样本量统计（总学生数、学校数）
- 性别分布分析
- 办学性质分布（公办vs民办）
- 办学规模分布（微规模、小规模、中规模、大规模）
- 数据质量检查与清洗

### 2. 整体学情画像
- 各指标平均分计算与对比
- 风险暴露率分析（低于60分的比例）
- 关键指标识别（高风险指标）
- 优势指标识别

### 3. 群体差异分析
- **性别差异**：男生vs女生各指标对比
- **办学规模差异**：不同规模学校学生表现对比
- **办学性质差异**：公办vs民办学校对比
- 差异显著性分析

### 4. 学校层面定位分析
- 学校层面环境-发展定位图（气泡图）
- 表现优秀学校识别（前5名）
- 表现欠佳学校识别（后5名）
- 学校差异显著性分析

### 5. 学生分层：四象限识别
- 以成长环境和学生发展中位数为划分标准
- 第一象限：优势发展型（环境好、发展好）
- 第二象限：逆境成长型（环境欠佳、发展好）
- 第三象限：潜力待发型（环境好、发展欠佳）
- 第四象限：重点关注型（环境欠佳、发展欠佳）
- 各象限学生人数统计与占比

### 6. 干预建议生成
- 针对不同象限学生的个性化干预建议
- 针对高风险指标的改进措施
- 针对学校层面的帮扶建议
- 长效机制建设建议

## 使用方法

### 快速开始

```bash
# 1. 准备数据文件
将数据文件放在 data/ 目录下

# 2. 执行分析脚本
python scripts/education_analysis.py --data data/学生指标得分清单.xlsx

# 3. 查看结果
在 output/ 目录查看生成的报告、图表和数据文件
```

### 详细使用

```bash
# 自定义输出目录
python scripts/education_analysis.py \
  --data data/学生指标得分清单.xlsx \
  --output custom_output

# 自定义指标前缀（适用于不同列名格式）
python scripts/education_analysis.py \
  --data data/my_data.xlsx \
  --env-prefix "环境_" \
  --dev-prefix "发展_" \
  --verbose

# 完整参数示例
python scripts/education_analysis.py \
  --data data/my_data.xlsx \
  --output my_analysis \
  --env-prefix "成长环境_" \
  --dev-prefix "学生发展_" \
  --verbose
```

### 新增参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--env-prefix` | 成长环境指标前缀 | `成长环境_` |
| `--dev-prefix` | 学生发展指标前缀 | `学生发展_` |

### 动态检测说明

脚本会自动检测以下内容（无需手动配置）：

1. **表头行位置** - 扫描前10行，查找包含指标前缀的行
2. **指标列名** - 根据前缀自动识别所有指标列
3. **分类值** - 从数据中提取实际的性别、办学规模、办学性质值
4. **四象限阈值** - 使用中位数动态计算
5. **改进建议** - 根据分析结果自动生成针对性建议

### 动态建议生成

v1.2 版本新增动态建议生成功能：

1. **自动生成**：根据分析数据自动生成针对性改进建议
   - 基于高风险指标生成优先干预建议
   - 基于四象限分析生成分层干预建议
   - 基于学校分析生成帮扶建议

2. **自定义建议**：支持传入自定义建议覆盖默认内容
   ```python
   analyzer = EducationDataAnalyzer(data_path)
   analyzer.run_analysis()
   
   # 自定义建议
   custom_recommendations = [
       ('一、优先干预措施', ['建议1', '建议2']),
       ('二、针对性改进措施', ['建议1', '建议2']),
       ('三、长效机制建设', ['建议1', '建议2'])
   ]
   analyzer.generate_report(recommendations=custom_recommendations)
   ```

3. **大模型重写接口**：`get_analysis_summary()` 获取分析摘要
   ```python
   # 获取分析结果摘要（供大模型参考）
   summary = analyzer.get_analysis_summary()
   # summary 包含：基本信息、指标分析、四象限分析、学校分析等
   # 可传递给大模型生成更专业的建议
   ```

### 四象限划分标准
默认使用**中位数**作为划分标准（动态计算），不再硬编码阈值：

默认60分以下为风险，可自定义：

## 技术细节

### 数据清洗流程
1. 移除表头行（多级表头处理）
2. 数值列类型转换
3. 缺失值处理（删除含缺失值的行）
4. 脏数据识别与清理（识别非数据行）

### 统计计算方法
- 平均分：`df[col].mean()`
- 风险暴露率：`(df[col] < threshold).sum() / len(df) * 100`
- 四象限划分：基于中位数二分法
- 学校层面聚合：`df.groupby('学校代码').agg()`

### 可视化规范
- 图表分辨率：300 DPI
- 中文字体：SimHei
- 颜色方案：蓝色系（#4472C4）、橙色系（#ED7D31）、红色系（#FF6B6B）
- 图表尺寸：根据内容自适应

## 最佳实践

### 数据准备
1. 确保数据包含完整的指标列
2. 检查数据质量（缺失值、异常值）
3. 确认列名与配置匹配

### 分析执行
1. 先进行数据概况检查
2. 关注高风险指标
3. 重点分析第四象限学生
4. 对比不同学校表现

### 报告使用
1. Word报告用于正式汇报
2. CSV数据用于深度分析
3. 图表用于可视化展示
4. 干预建议用于实际应用

2. **样本量要求**：建议样本量 > 100
3. **指标完整性**：至少包含4个成长环境指标和6个学生发展指标
4. **中文环境**：确保系统支持中文字体渲染
